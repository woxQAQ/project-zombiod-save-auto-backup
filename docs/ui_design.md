# Project Zomboid Save Backup Tool - UI/UX Design Document

## 1. 设计原则 (Design Principles)

*   **沉浸感 (Immersion)**：采用深色主题 (Dark Mode)，配色灵感来源于游戏 UI（黑、灰、红），与游戏氛围保持一致。
*   **极简主义 (Minimalism)**：核心功能（备份/恢复）必须一目了然，减少不必要的干扰元素。
*   **安全感 (Safety)**：所有的破坏性操作（如恢复、删除）都必须有明确的视觉警示和确认流程。
*   **即时反馈 (Feedback)**：操作成功、失败或进行中时，必须有清晰的状态提示。

## 2. 视觉风格 (Visual Style)

*   **背景色**：`#1a1a1a` (深灰，接近黑色)
*   **前景色/文字**：`#e0e0e0` (浅灰，高对比度)
*   **强调色 (Primary)**：`#d32f2f` (深红，代表生存/紧急，用于主要按钮)
*   **成功色 (Success)**：`#388e3c` (绿色，用于状态指示)
*   **警告色 (Warning)**：`#fbc02d` (黄色，用于确认弹窗)
*   **字体**：无衬线字体 (System Default / Inter / Roboto)，保持清晰易读。

## 3. 界面布局 (Layout)

应用采用 **左侧导航 + 右侧内容** 的经典布局，或 **顶部轻导航 + 单页应用** 结构。考虑到功能较少，推荐 **单页仪表盘 (Single Page Dashboard)** 结构，通过模态框 (Modal) 处理设置。

## 4. 详细组件设计 (Component Detail)

### 4.1 头部 (Header)
*   左侧：应用 Logo 和标题。
*   右侧：**设置齿轮图标**。点击弹出设置模态框。

### 4.2 存档选择器 (Save Selector)
*   **样式**：一个醒目的下拉菜单或标签页。
*   **内容**：显示检测到的存档名称（文件夹名）。
*   **元数据**：如果可能，显示该存档的最后修改时间。
*   **空状态**：如果未检测到存档，显示“未找到存档，请在设置中指定路径”的提示链接。

### 4.3 控制面板 (Control Panel)
*   **自动备份开关**：一个带文字的 Toggle Switch 组件。开启时显示绿色呼吸灯效果。
*   **立即备份按钮**：
    *   尺寸：大号 (Large)。
    *   颜色：强调色 (红色)。
    *   状态：点击后变为“备份中... (Spinner)”，并禁用点击。

### 4.4 备份历史列表 (History List)
*   **列表头**：无需表头，直接展示卡片列表。
*   **列表项 (Item Card)**：
    *   **左侧**：备份时间（高亮显示），下方小字显示“距今多久” (e.g., "2 hours ago")。
    *   **中间**：文件大小 (e.g., "1.2 GB")。
    *   **右侧操作区**：
        *   **恢复按钮 (Restore)**：蓝色或白色边框按钮。点击触发二次确认。
        *   **删除按钮 (Delete)**：垃圾桶图标，红色。仅鼠标悬停时显示，保持界面整洁。
*   **空状态**：显示“暂无备份记录”插画。

### 4.5 设置模态框 (Settings Modal)
*   **路径设置**：
    *   源目录输入框 + “浏览”按钮。
    *   备份目录输入框 + “浏览”按钮。
*   **策略设置**：
    *   自动备份间隔：滑块 (Slider) 或 下拉选择 (5min, 10min, 30min...)。
    *   保留份数：数字输入框 (默认 10)。
*   **底部**：保存按钮、取消按钮。

## 5. 关键交互流程 (Interaction Flows)

### 5.1 恢复备份 (Restore Flow)
1.  用户点击列表中的“恢复”按钮。
2.  **系统弹窗 (Warning Modal)**：
    *   标题：“确认恢复存档？”
    *   内容：“此操作将覆盖当前的 'MySurvival' 存档。系统将在恢复前自动创建一个临时快照以防万一。”
    *   按钮：“确认恢复 (红色)”、“取消”。
3.  用户点击确认。
4.  界面进入 **Loading 遮罩层**，显示“正在创建安全快照...”、“正在恢复数据...”。
5.  完成后，显示 Success Toast：“存档已恢复至 14:30 的状态。”

### 5.2 首次启动 (Onboarding)
1.  应用启动。
2.  检查是否能找到默认路径。
3.  **Case A (找到)**：直接进入主界面。
4.  **Case B (未找到)**：
    *   显示欢迎页：“欢迎使用 PZ 备份助手。我们无法自动定位您的游戏存档，请手动指定。”
    *   提供“选择文件夹”按钮。
    *   配置完成后进入主界面。

## 6. 响应式与适配
*   **最小尺寸**：800px x 600px (确保在小屏幕笔记本上可用)。
*   **自适应**：列表区域高度自适应，确保可以通过滚动查看历史记录。
